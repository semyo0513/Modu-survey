## 1. 프로젝트 개요

### 1.1 배경 및 목적
- 구글 폼의 한계(제작 불편, 단조로운 통계 시각화)를 극복하는 자체 설문 플랫폼 구축
- **빈도분석, 연관어분석, 교차분석** 등 데이터마이닝 기법을 내장한 시각화 대시보드 제공
- 학교 단위 운영에 최적화: 교사(회원)가 설문을 만들고, 학생(비회원)이 학교·설문명 선택으로 간편 참여
- 무료 인프라(GitHub Pages + Google Apps Script + Google Sheets)로 운영 비용 0원

### 1.2 사용자 역할 정의

| 역할 | 인증 방식 | 주요 기능 |
|------|-----------|-----------|
| **설문 제작자 (회원)** | 회원가입 / 로그인 (이메일 + 비밀번호) | 설문 생성·수정·마감, 결과 대시보드 열람, CSV 내보내기 |
| **응답자 (비회원)** | 인증 불필요 | 학교 선택 → 설문조사 이름 선택 → 응답 제출 |
| **시스템 관리자** | GAS 소유 계정 | 시트 직접 관리, 스크립트 배포 |

## 3. ★ DB 자동 초기화 설계 (개정 2)

### 3.1 동작 원리
웹앱(GAS) 최초 실행 시 `initializeDatabase()`가 호출되어 **필요한 시트가 없으면 자동 생성**하고 헤더·서식·검증 규칙까지 구성한다. 이후 실행부터는 검사만 하고 건너뛰므로 성능 부담이 없다.

```javascript
// Code.gs — 모든 doGet/doPost 진입 시 1회 보장
function ensureDatabase_() {
  const props = PropertiesService.getScriptProperties();
  if (props.getProperty('DB_INITIALIZED') === 'v2') return; // 이미 구성됨

  const lock = LockService.getScriptLock();
  lock.waitLock(30000);
  try {
    const ss = SpreadsheetApp.getActiveSpreadsheet();
    SHEET_SCHEMAS.forEach(schema => {
      let sheet = ss.getSheetByName(schema.name);
      if (!sheet) sheet = ss.insertSheet(schema.name);
      if (sheet.getLastRow() === 0) {
        sheet.getRange(1, 1, 1, schema.headers.length)
             .setValues([schema.headers])
             .setFontWeight('bold')
             .setBackground('#1a73e8')
             .setFontColor('#ffffff');
        sheet.setFrozenRows(1);
      }
    });
    props.setProperty('DB_INITIALIZED', 'v2');
  } finally {
    lock.releaseLock();
  }
}
```

### 3.2 자동 생성되는 시트 스키마

```javascript
const SHEET_SCHEMAS = [
  { name: 'Users',     headers: ['userId','email','passwordHash','salt','name','schoolName','role','createdAt','lastLoginAt','status'] },
  { name: 'Surveys',   headers: ['surveyId','ownerId','schoolName','title','description','status','startAt','endAt','allowMultiple','themeColor','createdAt','updatedAt'] },
  { name: 'Questions', headers: ['questionId','surveyId','order','type','title','required','optionsJson','gridJson','validationJson'] },
  { name: 'Responses', headers: ['responseId','surveyId','submittedAt','deviceType','durationSec','signatureUrl'] },
  { name: 'Answers',   headers: ['answerId','responseId','surveyId','questionId','questionType','value','valueRow','valueCol','textRaw'] },
  { name: 'Schools',   headers: ['schoolId','schoolName','region','createdAt'] },
  { name: 'Config',    headers: ['key','value','description'] }
];
```

| 시트 | 역할 | 비고 |
|------|------|------|
| `Users` | 회원 계정 | 비밀번호는 **SHA-256(salt+pw) 해시만 저장** |
| `Surveys` | 설문 메타데이터 | `schoolName` 필드로 학교별 필터링 |
| `Questions` | 문항 정의 | `type`에 12종 문항 타입, 그리드는 `gridJson`에 행/열 정의 |
| `Responses` | 응답 헤더(1건 = 1제출) | 서명 이미지는 Drive URL 참조 |
| `Answers` | 응답 상세 (**롱포맷**) | 그리드 응답은 `valueRow`/`valueCol`로 저장 |
| `Schools` | 학교 마스터 | 응답자 첫 화면의 학교 선택 목록 소스 |
| `Config` | 시스템 설정 | 세션 만료시간, 불용어 사전 버전 등 |

> **롱포맷 유지 이유**: 문항 추가·삭제 시 스키마 변경이 불필요하고, 교차분석·연관어분석 시 pivot이 용이함.

## 4. 사용자 흐름 설계 (개정 1)

### 4.1 설문 제작자 흐름 (회원)

```
회원가입(이메일·비밀번호·이름·학교명)
   → 이메일 중복 검사 → 가입 완료
로그인 → 토큰 발급(24h 유효)
   → 내 설문 목록 (내가 만든 설문만 표시)
   → [새 설문 만들기]
        → 설문 정보 입력 (제목, 설명, 학교명 자동입력, 기간)
        → 문항 빌더 (드래그 정렬, 12종 타입, 미리보기)
        → 저장 → 상태: draft → [게시] → 상태: open
   → [결과 보기] → 데이터마이닝 대시보드
   → [마감] → 상태: closed
```

### 4.2 응답자 흐름 (비회원 — 2단계 선택 진입)

```
웹앱 접속 (로그인 불필요)
   → STEP 1: 학교 선택 (드롭다운 + 자동완성 검색)
   → STEP 2: 해당 학교의 진행 중(open) 설문조사 목록 표시
              → 설문조사 이름 선택
   → 설문 응답 화면 (문항 순차 진행, 진행률 표시)
   → 서명 문항 시 터치패드 서명
   → 제출 → 완료 화면 ("응답이 저장되었습니다")
```

- 직접 링크 접근도 지원: `?school=OO중학교&survey=SRV-0001` 쿼리 파라미터로 STEP 1·2 건너뛰기 가능 (QR코드 배포용)
- `allowMultiple=false`인 설문은 `localStorage` 응답 이력으로 중복 제출 1차 차단

### 4.3 화면(페이지) 목록

| 페이지 | 파일 | 접근 권한 |
|--------|------|-----------|
| 응답자 진입 (학교/설문 선택) | `index.html` | 전체 공개 |
| 설문 응답 | `respond.html` | 전체 공개 |
| 회원가입 | `signup.html` | 전체 공개 |
| 로그인 | `login.html` | 전체 공개 |
| 내 설문 목록 | `admin/dashboard.html` | 회원 |

| 결과 대시보드 | `admin/results.html` | 회원(소유자) |

## 5. 인증 설계

### 5.1 회원가입 / 로그인

```
[가입] 이메일 + 비밀번호 → salt 생성 → SHA-256(salt + pw) 해시 저장
[로그인] 해시 대조 → 성공 시 UUID 토큰 발급
        → CacheService에 token:userId 저장 (TTL 24시간)
[요청] 모든 제작자 API에 Authorization 토큰 첨부 → 서버에서 검증
[로그아웃] 캐시에서 토큰 제거
```

### 5.2 보안 원칙
- 평문 비밀번호는 어디에도 저장·로그하지 않음
- 결과 조회 API는 **토큰 소유자 = 설문 소유자(ownerId)** 일치 검증
- 응답 제출은 비회원 허용하되, surveyId 유효성·open 상태·기간 검증
- `LockService`로 동시 쓰기(행 append) 충돌 방지
- GAS Web App CORS 특성 대응: `Content-Type: text/plain` POST + `redirect: follow`

## 6. ★ 응답 모델(문항 타입) 설계 (개정 3)

### 6.1 지원 문항 타입 12종

| # | 타입 코드 | 이름 | 입력 UI | Answers 저장 방식 |
|---|-----------|------|---------|-------------------|
| 1 | `radio` | 단일 선택 | 라디오 버튼 카드 | `value` = 선택지 |
| 2 | `checkbox` | 복수 선택 | 체크박스 카드 | 선택지당 1행 (멀티행) |
| 3 | `dropdown` | 드롭다운 | select + 검색 | `value` = 선택지 |
| 4 | `text` | 단답형 | 한 줄 입력 | `textRaw` |
| 5 | `textarea` | 서술형(장문) | 여러 줄 입력 | `textRaw` (→ 텍스트마이닝 대상) |
| 6 | `grid_radio` | 객관식 그리드 | 행×열 라디오 행렬 | 행당 1행: `valueRow`+`valueCol` |
| 7 | `grid_checkbox` | 체크박스 그리드 | 행×열 체크 행렬 | 체크당 1행: `valueRow`+`valueCol` |
| 8 | `scale` | 선형 척도 | 1~N 슬라이더/버튼 | `value` = 숫자 |
| 9 | `rating` | 별점/이모지 평가 | ★ 또는 😍😐😡 탭 | `value` = 숫자 |
| 10 | `rank` | 순위 매기기 | 드래그 정렬 | 항목당 1행: `value` = 순위 |
| 11 | `signature` | **터치패드 서명** | Canvas 서명 패드 | `signatureUrl` = Drive 이미지 URL |
| 12 | `date` | 날짜/시간 | date/time picker | `value` = ISO 문자열 |

### 6.2 터치패드 서명 상세 설계

```
[프론트] signature_pad.js
  - <canvas> 요소, 터치·마우스·스타일러스 입력
  - 지우기 / 다시 쓰기 버튼, 빈 서명 제출 방지
  - 제출 시 toDataURL('image/png') → Base64 문자열

[백엔드] GAS
  - Base64 디코딩 → Blob 생성
  - DriveApp: "SurveyMine_Signatures/{surveyId}/" 폴더에 PNG 저장
  - 공유 설정: 링크 소유자만 열람 (개인정보 보호)
  - Responses.signatureUrl에 파일 URL 기록

[용량 정책]
  - 캔버스 300×150px, PNG 평균 5~15KB
  - GAS POST 페이로드 제한(50MB) 대비 충분한 여유
```

### 6.3 그리드(행렬형) 문항 상세 설계

```json
// Questions.gridJson 예시 — "학교 생활 만족도"
{
  "rows": ["수업", "급식", "시설", "동아리"],
  "cols": ["매우불만", "불만", "보통", "만족", "매우만족"],
  "colScores": [1, 2, 3, 4, 5]
}
```
- 모바일에서는 행렬 대신 **행 단위 카드 스와이프 UI**로 자동 전환 (반응형)
- 대시보드에서 그리드 응답은 **히트맵 + 행별 평균 점수 막대**로 시각화

### 6.4 문항 빌더 UX
- 문항 타입 팔레트에서 클릭/드래그로 추가
- 실시간 미리보기 (응답자 화면과 동일 렌더링)
- 필수 여부, 선택지 편집, 그리드 행·열 편집, 유효성 규칙(`validationJson`: 글자 수, 숫자 범위 등)
- 문항 순서 드래그 정렬 (`order` 필드 갱신)

## 7. API 설계 (GAS Web App)

모든 요청: `POST {WEB_APP_URL}` / Body: `JSON.stringify({ action, token?, payload })`

| action | 인증 | 설명 |
|--------|------|------|
| `auth.signup` | — | 회원가입 (이메일, 비밀번호, 이름, 학교명) |
| `auth.login` | — | 로그인 → 토큰 반환 |
| `auth.logout` | 토큰 | 세션 종료 |
| `schools.list` | — | 학교 목록 (응답자 STEP 1) |
| `surveys.listBySchool` | — | 특정 학교의 open 설문 목록 (응답자 STEP 2) |
| `surveys.listMine` | 토큰 | 내가 만든 설문 목록 |
| `surveys.create` / `update` / `setStatus` | 토큰 | 설문 CRUD, draft→open→closed |
| `questions.bulkSave` | 토큰 | 문항 일괄 저장 |
| `survey.get` | — | 응답용 설문+문항 조회 (open 상태만) |
| `response.submit` | — | 응답 제출 (서명 Base64 포함 가능) |
| `results.get` | 토큰(소유자) | 전체 응답 데이터 반환 → 클라이언트 분석 |
| `results.exportCsv` | 토큰(소유자) | CSV 다운로드 |

> 분석 연산(빈도·연관어·교차)은 **100% 클라이언트에서 수행** — GAS 6분 실행 제한·쿼터 회피, 대시보드 인터랙션 실시간 유지.

## 8. 데이터마이닝 & 시각화 설계

### 8.1 분석 모듈

| 분석 | 대상 문항 | 방법 | 시각화 |
|------|-----------|------|--------|
| 빈도분석 | 선택형·척도·그리드 | 단순/상대 빈도, n-gram(서술형) | 막대·도넛·파레토·워드클라우드 |
| 연관어분석 | 서술형 | 형태소 분리(간이 토크나이저) + 동시출현 행렬 + Jaccard/PMI | **네트워크 그래프** (vis-network) |
| 교차분석 | 선택형 × 선택형 | 교차표 + 카이제곱 검정 | 히트맵, 누적 막대 |
| 그리드 분석 | grid_* | 행별 평균·분포 | 히트맵 + 평균 막대 |
| 장바구니 분석 | 복수선택 | 지지도·신뢰도·향상도 | 연관 규칙 테이블 |
| 감성분석(선택) | 서술형 | KNU 감성사전 기반 | 감성 도넛·추이 |
| 응답 추이 | 전체 | 시간대별 제출 수 | 라인 차트 |

### 8.2 대시보드 구성 (results.html)
- 상단 KPI 카드: 총 응답 수, 완료율, 평균 소요 시간, 학교명
- 문항별 자동 시각화 (타입에 맞는 차트 자동 매핑)
- 서술형 탭: 워드클라우드 ↔ 연관어 네트워크 ↔ 원문 목록 전환
- 필터: 기간, 그리드 행, 선택지 조건 → 전체 차트 실시간 갱신
- 서명 문항: 서명 수집 건수만 표시 (이미지는 개별 응답 상세에서 열람)

## 11. 리스크 및 대응

| 리스크 | 영향 | 대응 |
|--------|------|------|
| GAS 쿼터 (URL Fetch/일, 실행시간 6분) | 대량 응답 시 지연 | 분석은 클라이언트 수행, 쓰기는 append 최소화 |
| 시트 500만 셀 한도 | 장기 운영 시 한계 | 설문별 아카이브 시트 분리, 연 단위 스프레드시트 분할 |
| 서명 이미지 용량 | Drive 용량 증가 | 캔버스 크기 제한(300×150), PNG 압축, 마감 후 일괄 정리 옵션 |
| 동시 제출 충돌 | 행 유실 | LockService + 재시도 로직 |
| 비밀번호 유출 | 계정 탈취 | salt+SHA-256 해시, 평문 미저장, 로그인 시도 제한 |
| CORS 오류 | API 통신 실패 | text/plain POST + redirect follow 규약 문서화 |
